Instructions

This part is about improving the combat mechanics in the game:

  • Adding more moves or actions possible
  • Adding more effects or spells
  • Adding inventory and item management
Better Combat Options
Requirements Difficulty Mod file
Base Game & Dynamic Characters Easy better_combat_options.ts
  • The combat options must be displayed each turn
  • Each option can be entered with the corresponding number AND the name (case-insensitive)
  • The option “Escape” must be added. It stops the fight (you get to decide the repercussions of fleeing).
  • The option “Protect” must be added. It halves the received damage (see Damage Modifiers section)
More Statistics
Requirements Difficulty Mod file
Better Combat Options Easy more_statistics.ts
  • The def is used to reduce physical attack damage (see Damage Modifier, players.json, enemies.json and bosses.json)
  • The res is used to reduce magical attack damage
  • The spd is used to decide of the turn order. Having more speed means playing before. The fastest also has a chance to dodge equal to his speed minus the opponent's speed. For instance, if Link has 10 spd and his opponent 6 spd, Link will play first and will have 4% chance to dodge the enemy's attacks.
  • The luck is used to calculate the chances of a critical strike that deals double damage (see Damage Modifier)
Inventory
Requirements Difficulty Mod file
Better Combat Options Hard inventory.ts
  • The player can receive a random item at the end of each fight (see Rarity section)
  • The dropped items can be stored in an inventory
  • The inventory can be displayable
  • The items of the inventory can be consumed and removed from the inventory
Magic Skills
Requirements Difficulty Mod file
Better Combat Options Intermediate magic_skills.ts
  • “Heal” must be replaced by “Skills” in the main combat options
  • “Skills” display the list of the learned skills
  • Each playable character can use “Cheat Heal”, “Cheat Restore” and “Cheat Fireball”
  • Each playable character can use its class' skills (see skills.json).
  • The “Skills” must use the spells.json to calculate mana cost and damage
  • The spells.json effects “Heal” and “Restore” must be coded
    • The “Heal” effect heals the given amount of hp
    • The “Restore” effect restores the given amount of mana
  • If the character doesn't have enough mana, he can't cast the spell
Team Combat
Requirements Difficulty Mod file
Better Combat Options Hard team_combat.ts
  • The fights that were previously 1 versus 1 are now 4 versus 4
  • At the beginning of the game, instead of having 1 random playable character, you receive 4 random playable characters
    • You can't have the same character twice
  • At the beginning of each non-boss fight, 4 enemies appear instead of 1
  • Each turn, every character gets to play and attack a specific target
  • Each enemy attacks a random character
